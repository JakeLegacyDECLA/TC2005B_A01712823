Preguntas de la actividad

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    Internet es la estructura global de redes interconectadas que utilizan el protocolo TCP e IP, mientras que la World Wide Web es un servicio que funciona sobre internet y permite acceder a documentos interconectados mediante HTTP y navegadores web.
    (Internet Society. (s.f.). What is the Internet? https://www.internetsociety.org/internet/what-is-the-internet/)

  2. ¿Cuáles son las partes de un URL?

    En un URL tenemos el protocolo (HTTPS), el dominio (www.ejemplo.com), el puerto (8808), la ruta (/ruta/pagina.html), un query string (?clave=valor), y el fragmento (#seccion).
    (Berners-Lee, T., Fielding, R., & Masinter, L. (2005). RFC 3986: Uniform Resource Identifier (URI): Generic Syntax. Internet Engineering Task Force. https://doi.org/10.17487/RFC3986)

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    GET: Solicita recursos del servidor, HEAD: Igual que GET pero devuelve encabezados, POST: Envía datos al servidor, PUT: Reemplaza un recurso, PATCH: Modifica parcialmente un recurso y DELETE: elimina un recurso.
    (Fielding, R., Nottingham, M., & Reschke, J. (2022). RFC 9110: HTTP Semantics. Internet Engineering Task Force. https://doi.org/10.17487/RFC9110)

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML?

    Se debe usar el método POST ya que envía los datos en el cuerpo de la solicitud, no en el URL (mayor seguridad).
    (Mozilla. (s.f.). HTTP request methods. MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods)

  5. ¿Qué método HTTP se utiliza cuando se accede a una página a través de un URL?

    Se utiliza el GET, pues está solicitando un recurso.
    (Mozilla. (s.f.). GET. MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/GET)

  6. ¿Qué significa el código HTTP 200?

    Significa que la respuesta del servidor fue exitosa.
    (Fielding, R., Nottingham, M., & Reschke, J. (2022). RFC 9110: HTTP Semantics. Internet Engineering Task Force. https://doi.org/10.17487/RFC9110)

  7. ¿Es responsabilidad del desarrollador corregir un error 404?

    Depende. Si es un enlace roto sí es responsabilidad del desarrollador; si el usuario escribió mal la URL, no necesariamente.
    (Mozilla. (s.f.). 404 Not Found. MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/404)

  8. ¿Es responsabilidad del desarrollador corregir un error 500?

    Sí, es un fallo interno del servidor o backend.
    (Mozilla. (s.f.). 500 Internal Server Error. MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/500)

  9. ¿Qué significa que un atributo HTML5 esté deprecated?

    Significa que ya no se recomienda su uso. Ejemplos: font, center, big y frame.
    (World Wide Web Consortium. (2014). HTML5 – A vocabulary and associated APIs for HTML and XHTML. https://www.w3.org/TR/html5/)

  10. ¿Cuáles son las diferencias principales entre HTML4 y HTML5?

    HTML5 incluye etiquetas semánticas como header, nav, section y article, soporte multimedia nativo y nuevas APIs.
    (World Wide Web Consortium. (2014). HTML5 differences from HTML4. https://www.w3.org/TR/html5-diff/)

  11. ¿Qué componentes tiene una tabla?

    table, thead, tbody, tfoot, tr, th y td.
    (Mozilla. (s.f.). HTML table basics. MDN Web Docs. https://developer.mozilla.org/en-US/docs/Learn/HTML/Tables/Basics)

  12. ¿Cuáles son los principales controles de una forma HTML5?

    input, textarea, select y button.
    (Mozilla. (s.f.). HTML forms. MDN Web Docs. https://developer.mozilla.org/en-US/docs/Learn/Forms)

  13. ¿Qué soporte HTML5 tiene el navegador que utilizas?

    Uso Google Chrome y tiene alto soporte para HTML5.
    (HTML5Test. (s.f.). The HTML5 test – How well does your browser support HTML5? https://html5test.com/)

Sobre el ciclo de vida de los sistemas de información

  1. ¿Cuál es el ciclo de vida?

    Planeación, Análisis, Diseño, Implementación, Operación y Mantenimiento.
    (Guerrero, A. C., & Guerrero, A. C. (2015, July 20). Ciclo de vida de un sistema de información. Gestiopolis. https://www.gestiopolis.com/ciclo-de-vida-de-un-sistema-de-informacion/)

  2. ¿Cuál es el ciclo de desarrollo?

    Recolección de requisitos, Análisis, Diseño, Desarrollo, Pruebas, Implementación y Mantenimiento.
    (IEEE. (2018). ISO/IEC/IEEE 12207:2017 Systems and software engineering — Software life cycle processes. IEEE Standards Association.)
